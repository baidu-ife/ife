缠绕的内容和样式
样式和内容的分离实践

还有半个小时下班，让我喝口水平复下被前端技术虐哭的心情。

先说说html吧。常用标签自然不用细说，可是什么时候该用什么标签却经常迟疑。唉？我应该用div呢还是直接img呢？应该把a标签的display属性变成block呢还是inline-block呢，应该把这些部分归为整体呢还是应该把这部分归为整体呢等等等等。

开始写html文件的时候经常会把布局考虑进去，比如about页面中，看上去似乎分为左右两部分的简介。实现了之后怎么都觉得不对劲，为什么我把语义上是一个整体的东西因为形式而分离开了？这样做是不是有违样式和内容的分离初衷？

我想我获得的一个重要原则就是:

尽量不因为样式而影响HTML文件结构

如果恨不得每个元素都要单独设定样式的时候，就该想想是不是哪地方不对劲了。

ps1:永远也不要在行内元素中放置块状元素，这大概就是inline-block的意义吧。

PS2:另外提醒下自己要继续研究的问题：html5的那些标签还没用过啊。。。我什么时候该用，浏览器支持如何。大量的div是合适的吗？如何养成规范命名class或者id的习惯？

CSS
CSS虐我千百遍
我待CSS如初恋

手刷样式的日子，想起了当年痴心LaTeX排版的神奇体验。在那些日子里打下的一些本以为今后再也不会用到的概念再次从土里爬了出来= =

看，是很轻松的一件事。实践才发现自己很容易犯下不该犯下的错误。

一个简单的例子：当你想让元素距离其父元素上边一定距离时，你是应该怎么做呢？

  1. 使用父元素padding
  2. 使用子元素padding
  3. 使用子元素margin
  4. 使用子元素行高
  5. 混在一起做

我听见Larry Paul的声音从天上来，清晰而含糊

做一种事情不只一种方法，也不应该只用一种方法

如果你接手一个选择方案5的项目，你该如何想呢？WTF

还有其它超乎意料的东西：

  • 意外被css的选择优先级坑了
  • inline-block的元素间竟然有空格间隙
  • text-align: justify不是像我想的这么简单就能用
  • 究竟应该利用浮动还是绝对定位还是正常的文档流呢
  • 爱上了数学计算，特别是跟字体行高混合边距打交道时
  • input[type="submit"]标签在火狐中默认不是box-sizing:content-box属性。。。
  • 什么时候合并垂直的margin
  • 因为class名称问题一不小心样式相互影响了
  • 浏览器默认样式不一样
  • 当我写这篇文章时发现list的标号超出div了。。。
  • ...

真想说珍爱生命，远离前端。但它美的让人心碎。
